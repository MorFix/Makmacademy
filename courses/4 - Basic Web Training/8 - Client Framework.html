Introduction

After manipulating the DOM all by yourself, you'll learn a new framework – Angular JS/React (based on your project, ask your tutor).

AngularJS is an MVC framework by Google, its purpose is to allow easier SPA (Single Page Application) development in JS using MVC concepts.

React is a project by Facebook, a javascript library for building user interfaces

Before starting the exercise, take a few minutes to go over the basic tutorial.

In this exercise you will go back to our mail application and make it actually do something.

In addition, you will make it beautiful using Material-Design packages

Requirements

Your mail application will now live inside the client folder in the repository.

Your job is to transform your non-working mail application to a working application, using the provided REST server. All communications with the server will use JSON.

Your mail will support the following actions:

  • View incoming mail
    • View a list of all incoming mail
    • Viewing a specific mail
  • View outgoing mail
    • View a list of all outgoing mail
    • Viewing a specific mail
  • Send new mails, the new mails will be shown in the outgoing mail

Api

The provided server works with a classic REST api.

Incoming Mails - /incoming

  • GET / - Returns all incoming mails, long mail content will be trimmed.
  • GET /:id - Returns all data of a specified mail, long content is NOT trimmed.
  • DELETE /:id - Deletes a specified mail.

Outgoing Mails - /outgoing

  • GET / - Returns all outgoing mails, long mail content will be trimmed.
  • GET /:id - Returns all data of a specified mail, long content is NOT trimmed.
  • POST / - Sends a new mail, the mail should be a JSON object conforming to the mail structure without an ID.
  • DELETE /:id - Deletes a specified mail.
