Cookbook

Congratulations on completing the basic .NET training!

Now you will write a shared cookbook application.

1. Main Concept

The application will have a database of recipes, each recipe will have information regarding its ingredients and directions for making it.

Users will be able to register and log in, once the user is logged in they will be able to create recipes, update and delete their own recipes and save their favorites recipes.

2. Detailed Requirements

2.1 Recipe

A recipe is a collection of ingerdients with the steps for making it and general data about it like a category and a name.

Users can see all the recipes in the database, they can edit only their recipes.

The application should support filtering by categories, searching by name and by ingredients.

2.2 Ingredients

An ingredient represents an item required for making recipes, it may be used for a number of different recipes.

Recipes will contain a list of the required ingredients, each with the amount required for the recipe.

Users will be able to view all recipes containing a given ingredient, for example, look for all recipes containing broccoli as an ingredient.

2.3 Instructions

A single step in a recipe, a recipe is composed of a list of instructions in a fixed order.

Instructions may have additional information about them, like the average time for this step, included ingredients and more.

2.4 User

Users will register before visiting the application, registration requires a username and a password.

After logging in users can see the entire database and enjoy our application. If they find a recipe they like they can mark it as favorite and view their favorite recipes later.

4. Api

In this exercise you will work with external server at http://localhost:9754/.

To launch the server, download the attachment, extract it to C:\ and then run the following commands in a CMD AS ADMIN:

  • sc create "CookbookServer" binPath= "C:\CookbookServer\Cookbook.Server.exe"
  • sc start CookbookServer

In addition, you can download a common dll to help you communicate with the server.

Users - api/user

  • POST /login - Login to the app (LoginUserViewModel)
  • POST /register - Register to the app (NewUserViewModel)
  • GET /logout - Logout

Recpies - api/recipe

  • GET / - Get all the recipes
  • POST / - Insert recipe (RecipeViewModel)
  • PUT / - Update recipe (RecipeViewModel)
  • DELETE /:id - Delete recipe
  • POST /:id - toggle favorite (Use an empty StringContent as it's content)
