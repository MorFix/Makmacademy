Troubleshooting

Always commit

When you commit a change you can be certain you will never lose it. If you stash something and delete the stash you can be sorry later. If you reset uncommitted changes you also might suffer. But if you commit something. You can always find it later.

Reflog

The reflog is a great tool for going back in history. You can revert any change in the git tree. (undo branch reset, commit, rebase, pull, etc..). In order to go back in history open 2 git bashes:

In the first one write:

git reflog

And find the head position you want to go to, then write in the second bash (let's say for position 10 in the HEAD):

git reset --hard HEAD@{10}

In this example, I wanted to go back to when I commited "fix(sidenav-unread): show number of unread chats ...".

Now you can refresh git extensions and see you went back in time!

Even this action it-self is revertable using the the same technique.
