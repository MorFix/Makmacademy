Generic List Manager - Delegates & Events, Generics

In this exercise you will implement a generic doubly linked list named SuperList (Implemented using Node).

The list will implement the framework's generic IList interface to support the following:

  1. Add an item
  2. Remove an item
  3. Direct access to an item (indexer)
  4. Enumeration with ForEach

In addition to the IList methods, the list will implement methods that will perform a given action (Action/Func) on:

  1. A certain item
  2. All the items
  3. All the items that meet a given criteria (Predicate)

The list should expose an event of a change in the items (adding, removing), add the event information required (think of EventArgs).

Simple Bank

You will now use your list to create a simple bank management app.

The basic models will be SimpleAccount and VipAccount, both will have the following actions:

  • Deposit - Add a sum of money to the account's balance
  • Withdraw - Take a sum of money from the account's balance

Both of the account types will behave almost the same except for the Withdraw, where SimpleAccount isn't allowed to be in overdraft while the VipAccount is allowed.

The application will be a simple console application and will allow the following actions:

  1. Add an account
  2. Remove an account
  3. Deposit a certain amount to an account \ everyone
  4. Withdraw
  5. Every action you've performed needs to be logged using the logger you wrote previously

Our application needs to be pretty and in order to do that it will support changing the console's text color.

Implement ColorWriter class, this will be the only class that writes to the console and it will contain all the required actions for color support.

NOTE: The application should handle exceptions and errors properly and display correct messages to the user.
