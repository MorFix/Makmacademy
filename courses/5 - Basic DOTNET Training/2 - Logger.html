Logger

  • Implement a class named LogEntry, which will hold relevant information for a report. (Severity, Date and Time, Message)

  • Define an interface named ILog, with the following actions:

    • Write entry
    • Read entries (by date)
    • Clear the log

  • Write different implementations for your ILog interface:

    • CsvFileLog – Write to a CSV file.
    • EncryptedCsvFileLog – Write to a CSV file while encrypting your text (with an encryption of your choice).
    • EventLog – Write to the windows event log (so we could view it with the EventViewer)

  • Implement a general Logger class which will hold a certain ILog, and expose writing, reading and clearing actions.

  • Every application needs error handling. Implement the following custom errors:

    • NoLogDefinedException
    • LogIsFullException

  • Create a console application project, which receives as input the user's name, and then will present him with four options:

    • Blink your right eye
    • Blink your left eye
    • Administration
    • Exit

NOTES:

* Write every action performed by this application to the log.
* When "Administration" is selected, display every log entry written so far.
* In case of an exception, present the user with a proper message.
