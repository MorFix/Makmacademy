Part A

You will now build a simple bank system based on the MVC pattern, with the windows console as the UI.

Your models will be the following:

  • Account - Based on the same logic from exercise 2 (Write new models, both for Simple and VIP account)
  • User - Will contain the following information:
    • ID
    • Name

Every account can belong to more than a single user. Every user can have more than a single account.

The application will provide the following actions to the user:

  • List all users
  • List all accounts for a given user with their current balance
  • Deposit/Withdraw from a given account
  • Create a new user
  • Create a new account for a user
  • Delete users and accounts

* Only databases supported by the ADO namespace, DO NOT USE OleDB

NOTES:

  1. When you delete a user, you also need to delete his accounts, how do we accomplish this properly?
  2. Use the DataReader class in order to read data from the DB
  3. Implement converter classes to convert from DataReader data to your models
  4. Dispose of transactions properly and timely
